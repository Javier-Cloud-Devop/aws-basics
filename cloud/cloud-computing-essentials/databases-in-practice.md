# Databases in Practice - Amazon RDS

## Description 
In this lab, I explored AWS database services and created a relational database using Amazon RDS
The Objective was to understand how to deploy a managed database, enable high availability and configure read replicas

## Steps performed
- Reviewed AWS database offerings
- Launched an Amazon RDS database instance
- Configured a Multi-AZ deployment
- Enabled automated backups

## Architecture
- Primary Amazon RDS instance
- Multi-AZ deployment for high availability
- Read replica to improve read performance
- Resources distributed across multiple Availability Zones

## Key concepts
- Managed relational databases using Amazon RDS
- High Availability with Multi-AZ
- Read replicas for scaling read operations
- Automated backups manages by AWS

## Tool used
- Amazon RDS
- AWS Management Console (GUI)
